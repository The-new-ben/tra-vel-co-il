# tra-vel.co.il Operating Plan

Last updated: 2026-05-26

## Mission
Build a Hebrew money-first travel asset. Do not chase easy generic travel content first; prioritize commercial searches where affiliate, lead-gen, or service revenue can happen.

## Niche Hypothesis
Broad Hebrew travel can work only if it is organized around profitable verticals: travel insurance, vacation packages, organized tours, hotels, eSIM/SIM, car rental, flights, visas, and destination guides that push users toward offers.

## Front Page Direction
Hebrew RTL commercial hub, not a blog homepage. First viewport should immediately expose:
- destination/package search intent
- insurance and travel-service CTAs
- popular money pages
- lead capture for trip planning
- trust blocks and recent guides

## Theme Decision
Default path: custom lightweight WordPress theme coded in the repo and synced via UPress development/file-manager GitHub sync. Elementor may remain for legacy/editor pages, but core templates, front page, archive pages, money pages, schema, and lead forms should be code-first for speed, SEO control, and maintainability.

## CRM/Funnel
Install a simple CRM/lead pipeline similar to the Jus-Tice pattern:
- capture trip-planning, insurance, and package leads
- store source page, keyword intent, destination, budget, travel dates, phone/email
- route automated follow-up before any manual supplier work
- future payment/invoicing can use Grow/Green Invoice where relevant
